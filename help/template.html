1.  Introduction

With the exception of some specialty Web pages, this Web page is the template used for every Web page at Computer Help And Resources (CHAR). This Web page contains an example of almost every HTML structure/CSS presentation construct used throughout CHAR, including table of contents, Web page sectioning, paragraphs, notes, lists, tables, links, lists nested in lists, lists nested in notes, notes nested in lists, lists nested in tables, etc.

Traditionally, frames were used to display the same content on multiple Web pages. Frames, however, fell out of favor. As a substitute for frames, many authors use external JavaScript (.js) files with the document.write() method. The following .js files display the same content on multiple Web pages at CHAR:

.JS Files For Content At CHAR
.JS File Content
header.js Header for all Web pages except some specialty Web pages.
footer.js Footer for all Web pages except the site home page and some specialty Web pages.

With the exception of some specialty Web pages, every Web page at CHAR is valid HTML5 per the W3C Markup Validation Service (validator.w3.org). To confirm, click the W3C Valid HTML5 button near the bottom of each validating Web page, including this one.

With the exception of some specialty Web pages, every Web page at CHAR is styled by a single external style sheet which styles the entire Web page, including header, main body, and footer content. Regardless the source, all styles and style sheets at CHAR are valid CSS3 per the W3C CSS Validation Service (jigsaw.w3.org). To confirm, click the W3C Valid CSS button near the bottom of each Web page, including this one.

With the exception of some specialty Web pages, every Web page at CHAR includes the print friendly option. For the Web pages with the print friendly option, there are three Web page versions; 1.) Default, 2.) Print Friendly With Images, and 3.) Print Friendly Without Images. For all three Web page versions the same .html file is used. The only difference between the versions is the external style sheet that is used. For the Default version, the external style sheet is char.css. For the Print Friendly With Images version, the external style sheet is char_print_with_images.css. And for the Print Friendly Without Images version, the external style sheet is char_print_without_images.css. In each case, the single external style sheet styles the entire Web page, including header, main body, and footer content. Site-wide, it is much easier to have one .html file and three .css files representing each Web page version than it is to have three .html files representing each Web page version and one .css file. For example, if a Web site had 100 Web pages, the former would entail 103 Web documents (100 .html files + 3 .css files), while the later would entail 303 Web documents ((3 x 100) .html files + 3 .css files). The following are the external style sheets for the Web pages with the print friendly option at CHAR.

2.  TOC And Page Sections

Table of contents (TOC) and page sections mirror each other with respect to content, markup, and styles. For example, TOC section title corresponds to page section section title in that both are styled { font-size: 1.17em; font-weight: bold; }.

7.  Notes

Notes are for text that should be separated from the normal flow of text. There are two note types; 1.) normal notes, and 2.) important notes.

Notes (normal and important) consist of:

  1. A container: <div class="note"></div>.
  2. A marker that is floated left in the container. Currently, the marker is simply styled text:
    • Marker for normal notes: <div class="noteMarker noteMarkerNormal">i</div>.
    • Marker for important notes: <div class="noteMarker noteMarkerImportant">!</div>.
  3. Note content that is floated left in the container after the marker: <div class="noteContent">Note content here.</div>.
  4. Clear floating: <div class="clearBoth"></div>.

A normal note:

<div class="note">
    <div class="noteMarker noteMarkerNormal">i</div>
    <div class="noteContent">Normal note content.</div>
    <div class="clearBoth"></div>
</div>

An important note:

<div class="note">
    <div class="noteMarker noteMarkerImportant">!</div>
    <div class="noteContent">Important note content.</div>
    <div class="clearBoth"></div>
</div>

Note styles:

.note { margin: 0px 0px 18px 2px; }
.noteMarker { float: left; width: 22px; font-family: georgia, "times new roman", serif; font-size: 1.67em; font-weight: bold; font-style: italic; text-align: left; padding-top: 2px; } /* Requires simultaneous use of either .noteMarkerNormal or .noteMarkerImportant. */
.noteMarkerImportant { color: #882222; }
.noteMarkerNormal { color: #222288; }
.noteContent { float: left; width: 75%; font-size: 0.92em; background-color: #f7f7f7; border: 1px solid #aaaaaa; border-radius: 6px; box-shadow: 0px 1px 6px 0px #cccccc; padding: 6px; }
.clearBoth { clear: both; }

To convert a note from normal to important, or vice versa:

  1. Change the marker text from i to !.
  2. Change the marker class from noteMarkerNormal to noteMarkerImportant.

14.  Text Conventions

14.1.  Margin Left One Tick

To indent text "one tick" compared to paragraphs, use the p element classed class="marginLeftOneTick". The marginLeftOneTick class selector is assigned style:

.marginLeftOneTick { margin: 0px 0px 15px 20px; }

14.2.  Code

Code includes language keywords, identifiers, source code text, source code comments, etc. To indicate code, use the code element. The code element is assigned style:

code { font-family: "courier new", courier, monospace; }

To indicate a single word related to code, use the code element.

Single word related to code syntax:

<code>keyword</code>

Single word related to code examples:

For example, the color property is a characteristic of text, and the black value is a specific option of color that can be assigned to a property, such as color. The colon (:) in the declaration assigns the value to the property, thereby defining a style.

Paragraph using the p element not classed as reference.

To indicate a line or block of source code, use <p class="marginLeftOneTick"><code></code></p>. The marginLeftOneTick class selector is assigned style:

.marginLeftOneTick { margin: 0px 0px 15px 20px; }

Line or block of source code syntax:

<p class="marginLeftOneTick"><code>
selector { declaration block } /* This is a comment. */
selector { declaration block } /* Comments associated with lines and blocks of source code are treated as code, itself, and, therefore, are included along with the source code inside <p class="marginLeftOneTick"><code></code></p> tags. */
selector { declaration block } /* Comments use the comment syntax of the source code language. In this example, the source code is CSS. Therefore, the comment uses CSS comment syntax, not HTML or some other language comment syntax. */
selector { declaration block } /* Comment first letter first word Cap and end with period. */
</code></p>

To indicate quoted text, use <p class="marginLeftOneTick"><i></i></p>. The source is normal text. The marginLeftOneTick class selector is assigned style:

.marginLeftOneTick { margin: 0px 0px 15px 20px; }

Quoted text syntax:

<p class="marginLeftOneTick"><i>&quot;Quoted text&quot;</i> - Source</p>

Quoted text example:

"The primary sort of the declarations is by weight and origin: for normal declarations, author style sheets override user style sheets which override the default style sheet." - W3C CSS2 Recommendation

14.4.  Email Address And URI

To indicate an email address or URI as text, use the i element. To indicate an email address or URI as a hyperlink, do not use the i element. Instead simply use the a element classed either class="linkContent" or class="linkListItem" per 12.  Hyperlinks (above).

Email address and URI syntax:

<i>Email address as text</i>

<i>URI as text</i>

<a href="mailto:user@company.com" class="linkContent">Email address as mailto hyperlink</a>

<a href="http://char.learnwebcoding.com/" class="linkContent">URI as hyperlink in normal content</a>

<a href="http://char.learnwebcoding.com/" class="linkListItem">URI as hyperlink in an unordered list of one or more hyperlinks as list items</a>

Email address and URI example:

The email address as text is user@company.com.

The URI as text is http://char.learnwebcoding.com/.

The email address as mailto hyperlink is email.

The URI as hyperlink in normal content is http://char.learnwebcoding.com/.

  • The URI as hyperlink in an unordered list of one or more hyperlinks as list items is http://char.learnwebcoding.com/.

14.6.  Object To Click

To indicate an object to click, use the span element classed class="userInput". To indicate a sequence of objects to click, separate the objects with the | character, which is also included in the span element classed class="userInput". "Click" objects, not "Press" objects. For tabs, buttons, etc, do not include the word "tab," "button," etc. The userInput class selector is assigned style:

.userInput { font-weight: bold; }

Object to click syntax:

<span class="userInput">Object to click</span>

<span class="userInput">First object to click | Second object to click | Third object to click</span>

Object to click examples:

Click OK.

Either accept the default and click Next, or click New, create one or more drives, select a drive for the installation of Windows 8.1, and click Next

For example, in Opera 12.17, click Menu | Settings | Preferences | Advanced | Content | Style Options.

14.7.  Keyboard Key To Press And String To Type

To indicate a keyboard key to press, use the span element classed class="userInput". To indicate a combination of keys to press simultaneously, separate the keys with the + character, which is also included in the span element classed class="userInput". "Press" key, not "Click" key. "Type" string, not "enter," "press," nor "click" string. The userInput class selector is assigned style:

.userInput { font-weight: bold; }

Keyboard key to press and string to type syntax:

<span class="userInput">Key to press</span>

<span class="userInput">Key to press+Key to press+Key to press</span>

<span class="userInput">String of text to type</span>

Keyboard key to press and string to type examples:

Press Enter.

Press Ctrl+Alt+Del.

Type fdisk /mbr and press Enter.

14.8.  Dialog Title And Text

To indicate a dialog title and text, use the b element. Always use the dialog title. Dialog text is optional. If use dialog text, separate dialog title from dialog text with the : character. "Dialog," not "dialog box." Use the same spelling, uppercase/lowercase letters, and punctuation as the dialog.

Dialog title and text syntax:

Dialog title

Dialog title: Dialog text

Dialog title and text examples:

The Windows Setup dialog appears.

The Windows Setup: Which type of installation do you want? dialog appears.

The Windows Setup: Where do you want to install Windows? dialog appears.
