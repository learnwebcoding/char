  • Error Message When You Start A Windows 7 Or Windows Vista-Based Computer After You Change The SATA Mode Of The Boot Drive: "STOP 0x0000007B INACCESSABLE_BOOT_DEVICE" (922976) (support.microsoft.com)
    i
    With some motherboards (e.g., my Intel DG33FB) and a bootable SATA hard drive, changing the BIOS SATA mode to AHCI before enabling the Windows Vista/7 default SATA AHCI controller drivers results in a reboot loop, not a STOP 0x0000007B error. Either way: 1.) restart the computer and return the BIOS SATA mode to the original value (likely IDE); 2.) restart the computer and perform the two registry modifications described to enable the Windows Vista/7 default SATA AHCI controller drivers; 3.) restart the computer and change the BIOS SATA mode to AHCI; 4.) restart the computer and let Windows Vista/7 detect and install the SATA AHCI controller, which is listed in Device Manager under IDE ATA/ATAPI controllers; and 5.) to update the SATA AHCI controller drivers, install/reinstall the latest motherboard chipset drivers.

3.2.  Free Diagnostic Utilities: Hard Drive Manufacturer

!
Hard drive manufacturers provide utilities, free for download from their Web sites, to detect and repair hard drive problems. However, because each hard drive manufacturer's hard drives are different, the functionality of these utilities is often limited to, or should be limited to, basic nondestructive tests when run against a different manufacturer's hard drives. Therefore, it is strongly recommended that you use the hard drive manufacturer's own utilities to detect, and especially to repair, hard drive problems.
